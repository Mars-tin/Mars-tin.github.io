Introduction

Definition

Multiple instance learning (MIL) is a subclass of weakly supervised learning problem that deals with training data arranged in sets, called bags. Supervision is provided only for entire bags, and the individual labels of the instances contained in the bags are not provided. Positive instances are called witnesses.

Formulation

In the classical binary supervised learning problem one aims at finding a model that predicts a value of a target variable, y∈{0,1}y∈{0,1} for a given instance x∈RDx∈RD.

In a MIL problem, however, instead of a single instance there is a bag of instances X={x1,⋯,xK}X={x1,⋯,xK}. The size KK could vary for different bags. A single binary label YY is associated with the bag and individual labels exist for the instances within a bag, i.e., {y1,⋯,yK}{y1,⋯,yK} and yk∈{0,1}yk∈{0,1} for k∈[K]k∈[K]. However, there is no access to those labels and they remain unknown during training.

Motivition

  • Weak annotations over groups of instances are easier to obtain
  • Some problems like medical imaging are natural MIL problems

Applications

  • Drug activity prediction
  • Image and video classification
  • Document classification
  • Sound classification

Challenges

  • Interpretability
  • Sparse bags (low witness rate)
  • Application dependent

Reading Reflections

Ambiguity Survey

Multiple Instance Learning: Algorithms and Applications
Boris Babenko
Technical Report, 2008. [Paper]

Ambiguity

  • Polymorphism Ambiguity:
    • Each instance is a distinct entity or distinct version of an entity.
    • Usually independent.
  • Part-whole Ambiguity:
    • all instances are part of a whole object.
    • Usually dependent.

[Comments-M] In our application context, islets should better be considered independent.

Assumption Survey

A review of multi-instance learning assumptions
James Foulds, Eibe Frank
The Knowledge Engineering Review, 2010. [Paper]

Assumtions

  • Standard Assumption: only a few special instances with a positive label can have any influence on the class label
    • All negative bags contain only negative instances;
    • All positive bags contain at least one positive instance.
  • Collective Assumtpion: all instances in a bag contribute equally to the bag’s label.
    • a bag is not a finite collection of fixed elements but a sample from an underlying population specific to that particular bag;
    • a bag can be modeled as a probability distribution over the instance space, and the observed instances were generated by random sampling from that distribution.

[Comments-M] In our application context, bags are donors and islets are instances, I think the standard assumption makes more sense.

Survey 2018

Multiple instance learning: A survey of problem characteristics and applications
Marc-André Carbonneau, Veronika Cheplygina, Eric Granger, Ghyslain Gagnon
Pattern Recognition, 2018. [Paper][Code]

Assumption Settings

  • Setting 1: the bag is negative if and only if all the instances in it are negative;
  • Setting 2: the bag is positive if at least θθ instances in it is positive;
  • Setting 3: the bag is positive if at least θθ instances belong to all predefined concepts (Example: desert, sea and beach);
  • Setting 4: the bags is a probabilistic distribution over instances.

[Comments-M] In our application context, setting 2 seems more appropriate.

Task Formulation

  • Classification
    • Bag level classification
    • Instance level classification
    • Multiple labels to bags (Example: Image classification)
  • Regression
    • Bag level regression
  • Ranking
  • Clustering

[Comments-M] In our application context, since we need to explain the results, the donor-islet level explainabilty must be done by instance level classification.
